# 🌲 TPA Monte Verde — Institutional Website

<p align="center">
  <img src="./assets/tpa-monte-verde/tpa-01-home-hero.png" width="90%" />
</p>

## 🧭 Overview

**TPA Monte Verde** is an institutional website created to explain the **Environmental Preservation Fee** system in Monte Verde, Camanducaia - MG.

The project was designed to offer a clear, elegant, and accessible experience for visitors, organizing public information about the fee, exemption rules, legislation, payment guidance, and visitor support.

The visual identity follows a nature-inspired direction, using soft colors, environmental imagery, rounded elements, and a clean institutional layout.

---

## 🎯 Project Goals

- Create a modern public-facing institutional website
- Explain the TPA clearly to visitors
- Organize legal, financial, and exemption information
- Improve communication between the public service and users
- Provide a responsive and accessible interface
- Build a visual identity connected to Monte Verde’s natural environment

---

## ✨ Main Features

### 🌿 Institutional Presentation

- Hero section with strong visual impact
- TPA Monte Verde branding
- Navigation menu
- Call-to-action buttons
- Nature-inspired visual language

### 🧾 Informational Sections

- Explanation of what the TPA is
- Purpose of the environmental fee
- Resource application information
- Infrastructure and sustainability highlights
- Visitor-oriented content structure

### ✅ Exemption Area

- Exemption explanation modal
- Groups eligible for exemption
- Deadline and request information
- Web link and app guidance
- Card-based layout for better readability

### 💳 Payment Guidance

- Payment modal
- External portal release information
- Clear visitor instructions
- Simple and direct communication

### 📊 Fee Table

- Vehicle category table
- UFM-based values
- Conversion information
- Notes and observations

### ⚖️ Legislation Area

- Decree summary
- Main article highlights
- Legal documentation access
- Structured legal explanation

### ❓ FAQ and Footer

- Frequently asked questions
- Visitor support information
- Quick navigation links
- Institutional footer

# 📊 Operations Panel — Monitoring Dashboard

<p align="center">
  <img src="./assets/painel-operacoes/painel-02-equipamentos-dashboard.png" width="90%" />
</p>

## 🧭 Overview

The **Operations Panel** is a web-based dashboard developed to centralize operational monitoring, equipment data, failure records, quick-access tools, AI-assisted consultation, and support resources.

The system was created to support daily monitoring routines, allowing teams to quickly understand operational status, access related platforms, register failures, consult equipment information, and manage users.

Its interface was designed with a dark professional visual identity, emphasizing contrast, fast reading, visual indicators, and operational clarity.

---

## 🎯 Project Goals

- Centralize monitoring and operational information
- Improve visibility of equipment status
- Reduce time spent switching between platforms
- Support internal team workflows
- Organize failures, equipment, and city data
- Create a modern and efficient operational dashboard
- Provide restricted access and permission control
- Add AI support for operational consultation and troubleshooting

---

## ✨ Main Features

### 🔐 Authentication and Access Control

- Login screen
- Restricted access
- User account flow
- Password change modal
- Admin and operator profiles
- Permission management interface

### 📡 Equipment Monitoring

- General equipment dashboard
- Online and offline counters
- City-based equipment cards
- Manufacturer filters
- Status indicators
- Real-time oriented layout
- Visual counters and operational metrics

### 🗂️ Contractual and Equipment Data

- City search
- Equipment listing
- Model and function information
- Local reference organization
- Structured data visualization
- Operational lookup interface

### ⚠️ Failures and Occurrences

- Failure registration form
- Category, priority, status, and manufacturer fields
- Active failures and history table
- Filters by city, status, manufacturer, and category
- Operational notes and observations
- Internal workflow support for issue tracking

### 🗺️ Operational Map

- Map-oriented monitoring area
- City and equipment filters
- Search structure
- Coordinate support
- Prepared for geographic visualization

### 🤖 Operational AI Area

- Query interface for operational information
- Knowledge-base oriented layout
- Quick prompt buttons
- AI-assisted support for internal consultation and troubleshooting

### 🔗 Operational Access Hub

The dashboard includes quick-access areas for operational environments and support systems, including:

- SPLTV
- SWM
- Perkons
- Velsis
- Pumatronix

These access hubs help reduce friction during daily monitoring routines.

### 👥 User Management

- User list
- Role display
- Admin/operator control
- Permission update actions
- Password reset interface

### 🧰 Technical Reference Area

- Support-oriented reference section
- Internal command organization
- Technical resources grouped by category

> ⚠️ Sensitive technical references, credentials, private commands, and restricted information should not be exposed in the public version of this portfolio.
