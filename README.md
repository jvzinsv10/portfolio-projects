# 🌲 Project 1 — TPA Monte Verde

## Overview

**TPA Monte Verde** is an institutional and informational website created to explain the **Environmental Preservation Fee** system in Monte Verde, Camanducaia - MG.

The project was designed to provide a clear and elegant digital experience for visitors, helping them understand:

- what the TPA is,
- how it works,
- who may request exemption,
- how the fee is calculated,
- the legal basis behind the process,
- and how to access supporting services.

The visual identity was designed to be modern, light, welcoming, and strongly connected to the environmental and touristic character of Monte Verde.

---

## Main Goals

- Present public information in a clear and accessible way
- Improve digital communication with visitors
- Organize legal and operational content into a friendly interface
- Provide guidance on payment, exemptions, and legislation
- Create a visually appealing and responsive institutional website

---

## Main Features

### Institutional structure
- Hero section with strong visual identity
- Nature-inspired interface
- Smooth and clean section flow
- Public-facing institutional presentation

### Informational content
- Explanation of what the TPA is
- Purpose and application of collected resources
- Sustainability and infrastructure focus
- Visitor-oriented communication

### Exemption area
- Explanation of who can receive exemption
- Groups organized visually in cards
- Practical information about deadlines and requests
- App and web link guidance

### Fee table
- Vehicle category pricing table
- Structured pricing explanation
- Legal and operational notes

### Legal information
- Legislation section with decree summary
- Organized highlights of the main articles
- Access to legal documentation

### FAQ and support
- Frequently asked questions
- Guidance for common visitor concerns
- Institutional footer with contact and navigation

### UX/UI highlights
- Responsive layout
- Cards and modal windows
- Structured visual hierarchy
- Consistent color palette
- Better readability and usability

# 📊 Project 2 — Operations Panel

## Overview

The **Operations Panel** is a web-based internal dashboard created to centralize operational monitoring, equipment status, support tools, data organization, and workflow visibility.

The project was built to improve day-to-day operational control by concentrating relevant information in one place, allowing teams to quickly identify equipment conditions, access supporting tools, review operational data, and manage internal routines more efficiently.

The interface was designed to provide a clean, fast, modern, and highly visual operational experience.

---

## Main Goals

- Centralize monitoring information in a single interface
- Improve visibility of operational status
- Support internal workflows
- Speed up access to related systems and tools
- Provide clearer organization for equipment and operational data
- Reduce friction in daily monitoring and support routines

---

## Main Features

### Authentication and access
- Login interface for controlled access
- Credential-based restricted entry
- User account creation flow
- Password change modal
- Admin/operator permission structure

### Equipment monitoring
- Central dashboard for equipment overview
- Status display by city
- Online and offline counters
- Equipment grouping and filtering
- Quick visual interpretation of operational conditions

### Contractual and registration data
- City-based data navigation
- Equipment listing
- Contract-related organization
- Search and lookup structure
- Model, location, and function visibility

### Failures and occurrences
- Failure/occurrence registration form
- Operational status control
- History and active failure area
- Filtering by city, manufacturer, category, and status
- Structured internal workflow for issue tracking

### Operational map
- Area designed for geographic visualization
- Equipment and failure map support
- Search and filtering area
- Prepared for map integration and spatial monitoring

### Operational AI
- Dedicated operational query interface
- Designed for consulting failures, equipment, and knowledge base
- Quick suggestion buttons
- Support for structured operational consultation flow

### Access hub for external/internal systems
The panel also centralizes access to several operational environments and support links, such as:

- SPLTV
- SWM
- Perkons
- Velsis
- Pumatronix

This improves agility and reduces the need to navigate through multiple disconnected environments.

### User and permission management
- User list
- Roles and permissions view
- Admin/operator controls
- Administrative actions interface

### Technical support structure
- Area for support-oriented tools
- Reference and command organization
- Faster operational access to technical resources

> ⚠️ Some highly sensitive technical views, commands, and operational references may be intentionally omitted from the public version of this portfolio.
